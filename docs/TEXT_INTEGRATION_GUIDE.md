# 文字页集成功能说明 v2.0

## 功能概述

**优化升级**：将视频的文字转录按照**每张图片对应的时间段**精确匹配，文字页紧跟在对应图片的**下一张**位置，确保时间线的精确匹配。

## 核心改进（v2.0）

### 1. 精确时间段匹配
- **旧版本**：文字段落在图片之前插入，按中间时间点简单匹配
- **新版本**：
  - 分析每张图片的持续时间段（从当前图片到下一张图片）
  - 提取该时间段内的所有文字段落
  - 文字页紧跟在对应图片**后面**

### 2. 智能时间段计算
```python
对于图片i（时间Ti）：
  起始时间 = Ti
  结束时间 = T(i+1)  # 下一张图片的时间
  匹配规则：文字段落的中间时间点在 [Ti, T(i+1)) 区间内
```

### 3. 文字页尺寸自适应
- 文字页大小**完全匹配**对应图片的尺寸
- 保持PDF阅读的视觉一致性
- 字体大小调整为36px（更适合内容展示）

### 4. 页面布局优化
- 标题显示时间范围：`文字转录 (MM:SS - MM:SS)`
- 减少边距和行间距，容纳更多内容
- 超出页面时显示"还有X个段落未显示"提示

## 匹配逻辑详解

### 时间段分析示例
```
图片1: 5.00s
  ↓ 持续时间段: [5.00s - 10.00s)
文字页1: [5.5s-6.8s] "xxx"  [7.2s-9.5s] "xxx"
  ↓
图片2: 10.00s
  ↓ 持续时间段: [10.00s - 18.00s)
文字页2: [10.5s-12.3s] "xxx"  [15.0s-17.5s] "xxx"
  ↓
图片3: 18.00s
  ↓ 持续时间段: [18.00s - 25.00s)
文字页3: [18.2s-20.1s] "xxx"  [22.5s-24.8s] "xxx"
```

### 匹配规则
```python
for 每张图片i:
    start_time = 图片i的时间
    end_time = 图片(i+1)的时间  # 如果是最后一张，则为视频结束时间
    
    for 每个文字段落:
        seg_middle = (seg_start + seg_end) / 2
        if start_time <= seg_middle < end_time:
            该段落属于图片i
```

## 常见问题

### Q: 为什么文字页在图片后面而不是前面？
A: 这样更符合阅读习惯——先看画面，再看对应的文字说明，就像看PPT一样。

### Q: 有些图片没有对应的文字页？
A: 正常现象。只有当图片持续时间段内有文字内容时，才会生成文字页。本次处理中有9张图片（7%）没有对应文字。

### Q: 文字页显示"还有X个段落未显示"怎么办？
A: 可以调小字体大小或行间距，或者在代码中实现多页文字功能。

### Q: 如何调整文字页的字体大小？
A: 修改 `create_text_page()` 函数的 `font_size` 参数（默认36）。
