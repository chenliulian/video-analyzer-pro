## v3.0 核心特性

### 1. 合并时间显示 ✅
**问题**: 之前每个段落都显示独立的时间标记，占用大量空间
**解决**: 只在标题下方显示整体时间范围

```

新版本：

```

### 2. 多列并列布局 ✅
**问题**: 单列布局浪费横向空间，导致内容溢出
**解决**: 自动检测内容量，使用双列布局

```

┌────────────┐          ┌──────┬──────┐
│ 文字内容1   │          │ 文字  │ 文字  │

└────────────┘          │ 文字  │ 文字  │
                        │ 内容4 │ 内容9 │
                        │ 文字  │ 文字  │

```

### 3. 空间利用优化 ✅
- 减小字体: 36px → 32px
- 减小边距: 60px → 50px
- 减小行间距: 15px → 12px
- 标题区域压缩: 更紧凑的布局

### 4. 智能布局算法 ✅
```python
if 内容行数 > 单列最大行数:
    使用双列布局
    每列显示约一半内容
else:
    使用单列布局
    充分利用页面宽度
```

## 常见问题

### Q: 为什么不显示每个段落的时间了？
A: 为了节省空间，显示更多内容。如需查看精确时间，可以对照原始转录文件。

### Q: 可以调整为三列吗？
A: 可以。修改 `num_columns = 3`，但需要注意图片宽度要足够大。

### Q: 如何恢复显示每段时间？
A: 修改代码，在 `all_text` 合并时保留时间信息，但会牺牲显示空间。

### Q: 双列布局会影响阅读顺序吗？
A: 不会。文字从左列到右列顺序排列，符合自然阅读习惯。
